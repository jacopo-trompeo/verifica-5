1) git status mostrail repository con tutto lo storico di operazioni sui branch. La staging area è lo spazio in cui git prepara un commit: con git add si agginge il file, con git commit si sposta il file al repository.
2)'git checkout <branch>' fa spostare in un altro branch specificato. La history è un grafo delle operazioni nel repository. La HEAD è il ramo più aggiornato del repository, poi ci sono gli altri branch che si aggiornano indipendentemente; un branch nasce da un altro ed è indipendente anche da quello.
3)Il repository remoto è uno spazio in cui più persone aggiornano un progetto comune con operazioni di pull e push; non ha una working directory, quindi è solo un mezzo per lavorare in comune. 'git clone <remote repository>' crea un'alrta directory locale, copando il contenuto dentro di esso; inoltre crea un collegamento al repository remoto.
